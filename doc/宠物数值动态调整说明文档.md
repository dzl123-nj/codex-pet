# 宠物数值动态调整说明文档

## 1. 概述

本系统是一个虚拟宠物养成系统，通过模拟真实宠物的生理和心理需求，实现宠物数值的动态调整。宠物拥有饥饿值、能量、清洁度、幸福感、健康值五大核心属性，这些属性会随时间自然衰减，并通过玩家的交互操作（喂食、清洁、睡眠、玩耍等）得到恢复或消耗。系统通过算法确保宠物数值在合理范围内波动，并根据属性值触发状态转换（如健康、生病、死亡等），同时提供经验值和等级成长机制。

## 2. 核心属性

宠物拥有以下核心属性，所有属性值范围均为 **0-100**：

| 属性 | 说明 | 初始值 |
|------|------|--------|
| **饥饿值（Hunger）** | 宠物的饱腹程度，过低会导致生病 | 50 |
| **能量（Energy）** | 宠物的精力水平，影响活动能力 | 50 |
| **清洁度（Cleanliness）** | 宠物的卫生状况，过低会增加生病概率 | 50 |
| **幸福感（Happiness）** | 宠物的情绪状态，影响互动效果 | 50 |
| **健康值（Health）** | 宠物的健康状况，降至0会死亡 | 50 |
| **经验值（Experience）** | 累积经验值，用于等级提升 | 0 |
| **等级（Level）** | 宠物等级，最高100级 | 1 |

## 3. 时间衰减机制

### 3.1 基本衰减规则

宠物属性会随时间自然衰减，衰减量与时间间隔成正比：

- **饥饿值衰减**：每小时衰减 2 点
- **能量衰减**：每小时衰减 3 点（睡眠状态除外）
- **清洁度衰减**：每小时衰减 2 点
- **幸福感衰减**：每小时衰减 1 点
- **健康值衰减**：每小时衰减 1 点（睡眠状态除外）

### 3.2 睡眠状态特殊规则

当宠物处于睡眠状态（status=0）时：
- **能量恢复**：每小时恢复 5 点（衰减系数+2）
- **健康值恢复**：每小时恢复 1 点
- **能量衰减停止**：睡眠期间能量不衰减
- **健康值衰减停止**：睡眠期间健康值不衰减

### 3.3 衰减计算公式

```
衰减量 = 衰减系数 × 经过小时数
经过小时数 = (当前时间 - 上次更新时间) / (1000 × 60 × 60)
```

**注意**：
- 最小时间间隔为1分钟（60秒），低于此值不触发衰减
- 最大衰减时间限制为72小时，防止长时间未登录导致属性归零

### 3.4 连锁衰减效应

当某些属性低于阈值时，会加速其他属性的衰减：

- **饥饿值 ≤ 15**：幸福感额外衰减每小时1点，健康值额外衰减每小时3点
- **清洁度 ≤ 15**：幸福感额外衰减每小时1点，健康值额外衰减每小时2点
- **能量 ≤ 10**（非睡眠）：幸福感额外衰减每小时1点，健康值额外衰减每小时1点
- **生病状态**：健康值额外衰减每小时3点

## 4. 交互操作影响

### 4.1 喂食（Feed）

**前置条件**：
- 宠物未死亡（status≠3）
- 宠物未在睡觉（status≠0）
- 饥饿值 ≤ 90（防止过度喂食）

**数值变化**：
- 饥饿值：+30
- 幸福感：+15

### 4.2 清洁（Clean）

**前置条件**：
- 宠物未死亡（status≠3）
- 宠物未在睡觉（status≠0）

**数值变化**：
- 清洁度：+40
- 健康值：+5
- 幸福感：+5

### 4.3 睡眠（Sleep）

**前置条件**：
- 宠物未死亡（status≠3）
- 宠物未在睡觉（status≠0）

**数值变化**：
- 能量：+50
- 幸福感：+5
- 经验值：+5
- 状态变为睡眠（status=0）

### 4.4 唤醒（Wake）

**前置条件**：
- 宠物未死亡（status≠3）
- 宠物正在睡觉（status=0）

**数值变化**：
- 状态变为清醒（status=1）

### 4.5 玩耍（Play）

**前置条件**：
- 宠物未死亡（status≠3）
- 宠物未在睡觉（status≠0）
- 能量 ≥ 15

**数值变化**：
- 幸福感：+25
- 能量：-15
- 饥饿值：-5
- 清洁度：-5
- 经验值：+15

### 4.6 聊天奖励（Chat Reward）

**数值变化**：
- 幸福感：+5
- 能量：-2（仅在清醒状态）
- 经验值：+8

### 4.7 使用道具（Use Item）

**前置条件**：
- 宠物未死亡（status≠3）
- 宠物未在睡觉（status≠0）

**数值变化**：
- 根据道具属性动态调整（饥饿值、幸福感、能量、清洁度、健康值）
- 经验值：根据道具奖励值增加

## 5. 状态转换逻辑

宠物拥有以下状态：

| 状态码 | 状态名称 | 说明 |
|--------|----------|------|
| 0 | 睡眠 | 宠物正在睡觉，能量恢复中 |
| 1 | 清醒 | 宠物正常活动状态 |
| 2 | 生病 | 宠物生病，属性衰减加速 |
| 3 | 死亡 | 宠物健康值归零，无法交互 |

### 5.1 状态转换规则

#### 5.1.1 死亡判定
当健康值 ≤ 0 时，宠物立即死亡（status=3）。

#### 5.1.2 生病判定
当宠物处于清醒状态时，系统根据以下因素计算生病概率：

**基础生病概率**：
- 健康值 < 40：生病概率 = 0.7 + (40 - 健康值) × 0.01
- 健康值 40-59：生病概率 = 0.15 + (60 - 健康值) × 0.0125
- 健康值 60-79：生病概率 = (80 - 健康值) × 0.005
- 健康值 ≥ 80：生病概率 = 0

**额外生病概率加成**：
- 饥饿值 < 30：+0.1
- 清洁度 < 30：+0.1
- 能量 < 15：+0.1

**生病概率计算**：
```
总生病概率 = min(1.0, 基础概率 + 额外加成)
```

#### 5.1.3 生病状态恢复
当宠物已经生病时，恢复概率为生病概率的1.5倍。如果随机数大于等于生病概率×1.5，则宠物恢复健康。

#### 5.1.4 睡眠自动唤醒
当宠物处于睡眠状态且能量恢复至100时，自动唤醒（status=1）。

### 5.2 随机数生成

系统使用伪随机数决定生病/恢复事件，随机数种子基于：
```
种子 = 宠物ID × 31 + 时间段（30分钟为单位）
```
这确保了：
- 同一宠物在30分钟窗口内结果一致
- 不同宠物有不同的随机序列
- 防止频繁刷新刷状态

## 6. 经验与等级系统

### 6.1 经验值获取

不同操作提供不同经验值：

| 操作 | 经验值 |
|------|--------|
| 喂食 | 10 |
| 清洁 | 10 |
| 睡眠 | 5 |
| 玩耍 | 15 |
| 聊天 | 8 |
| 使用道具 | 根据道具属性 |

### 6.2 等级提升规则

**升级所需经验值**：
```
升级所需经验 = 当前等级 × 100 + 50
```

**示例**：
- 1级升2级：需要 1×100 + 50 = 150 经验
- 2级升3级：需要 2×100 + 50 = 250 经验
- 10级升11级：需要 10×100 + 50 = 1050 经验

**等级上限**：100级

**升级机制**：
- 经验值累积到升级阈值时自动升级
- 升级后扣除所需经验值，剩余经验继续累积
- 支持连续升级（如果经验值足够）

## 7. 数值边界控制

所有属性值通过`clamp`函数限制在0-100范围内：

```java
private long clamp(long value) {
    return Math.max(0, Math.min(100, value));
}
```

**边界规则**：
- 最小值：0
- 最大值：100
- 任何计算结果都会被限制在此范围内
- 防止数值溢出导致逻辑错误

## 9. 数据记录

系统会记录所有属性变化，包括：
- 变化时间
- 属性类型（hunger、energy、cleanliness、happiness、health）
- 变化前数值
- 变化后数值
- 变化量
- 变化原因（自然衰减、喂食、清洁、睡眠、玩耍、聊天、使用道具等）

这些记录用于：
- 宠物状态追踪
- 用户行为分析
- 系统调试和优化

## 10. 注意事项

1. **时间同步**：系统基于服务器时间计算衰减，确保所有客户端时间一致
2. **状态优先级**：死亡状态优先级最高，其次是睡眠状态
3. **边界保护**：所有数值计算都经过边界检查，防止异常值
4. **随机性控制**：生病/恢复机制使用伪随机数，确保公平性和可预测性
5. **性能优化**：衰减计算采用批量处理，减少数据库操作次数
