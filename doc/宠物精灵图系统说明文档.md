# 宠物精灵图系统说明文档

## 1. 概述

本系统通过接入 hatch-pet 生成的像素风精灵图，为桌面宠物提供丰富的动画效果。精灵图系统与后端动作系统解耦，互不冲突。

### 1.1 架构分层

```
┌─────────────────────────────────────────────┐
│  后端动作系统 (pet_action)                    │
│  业务逻辑层：属性变化、气泡文案、冷却时间      │
│  开放集：可无限扩展新动作                      │
├─────────────────────────────────────────────┤
│  映射层 (animation_name 字段)                 │
│  每个后端动作 → 一个精灵图动画 key            │
├─────────────────────────────────────────────┤
│  精灵图动画系统 (pet_sprite_config)           │
│  视觉表现层：帧动画、循环/单次播放             │
│  封闭集：由 sprite sheet 决定，不可扩展        │
└─────────────────────────────────────────────┘
```

### 1.2 核心原则

- **后端动作是开放集**：可以随时添加新动作（如"跳舞"、"吃饭"、"洗澡"）
- **精灵图动画是封闭集**：由 hatch-pet 生成的 sprite sheet 决定，有多少就是多少
- **两者通过映射关联**：每个后端动作指定一个精灵图动画 key，多个动作可复用同一动画
- **无冲突**：新增后端动作只需选一个已有的 sprite 动画，不需要修改精灵图

## 2. 数据库表结构

### 2.1 pet_action（后端动作表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 动作ID |
| action_code | varchar | 动作编码（feed/clean/sleep/wake/play/chat） |
| action_name | varchar | 动作名称（喂食/清洁/睡觉/唤醒/玩耍/聊天） |
| animation_name | varchar | **精灵图动画 key**（waving/jumping/idle/running 等） |
| trigger_type | varchar | 触发类型：auto/user/status/timed |
| duration | bigint | 动作持续时长（毫秒） |
| cooldown | bigint | 冷却时间（秒） |
| hunger_change | bigint | 饱食度变化 |
| happiness_change | bigint | 心情变化 |
| energy_change | bigint | 精力变化 |
| cleanliness_change | bigint | 清洁度变化 |
| health_change | bigint | 健康变化 |
| experience_reward | bigint | 奖励经验 |
| bubble_text | varchar | 气泡文案 |

**关键字段：`animation_name`** — 这是连接后端动作和精灵图动画的桥梁。

### 2.2 pet_sprite_config（精灵图配置表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 配置ID |
| pet_type_id | bigint | 关联宠物类型 |
| sprite_image | varchar | sprite 图片路径 |
| cell_w | int | 每格宽度（px） |
| cell_h | int | 每格高度（px） |
| fps | int | 动画帧率 |
| actions | text | 动作定义 JSON |
| status | char | 状态：0=启用，1=停用 |

**actions JSON 格式：**
```json
[
  {"key": "idle",          "row": 0, "frames": 6, "loop": true},
  {"key": "running-right", "row": 1, "frames": 8, "loop": true},
  {"key": "running-left",  "row": 2, "frames": 8, "loop": true},
  {"key": "waving",        "row": 3, "frames": 4, "loop": false},
  {"key": "jumping",       "row": 4, "frames": 5, "loop": false},
  {"key": "failed",        "row": 5, "frames": 8, "loop": false},
  {"key": "waiting",       "row": 6, "frames": 6, "loop": true},
  {"key": "running",       "row": 7, "frames": 6, "loop": true},
  {"key": "review",        "row": 8, "frames": 6, "loop": false}
]
```

## 3. 动作映射关系

### 3.1 默认映射（建议）

| 后端动作 | animation_name | 精灵图动画 | 说明 |
|----------|---------------|-----------|------|
| feed（喂食） | waving | 挥手 | 宠物开心挥手 |
| clean（清洁） | waving | 挥手 | 复用挥手动画 |
| sleep（睡觉） | idle | 待机 | 睡觉时保持静止 |
| wake（唤醒） | jumping | 跳跃 | 醒来时跳跃 |
| play（玩耍） | jumping | 跳跃 | 玩耍时跳跃 |
| chat（聊天） | review | 环顾 | 聊天时四处张望 |
| 空闲 | idle | 待机 | 默认状态 |
| 移动 | running / running-right | 跑步 | 桌面漫游时 |
| 生病 | failed | 失败 | 健康值过低时 |
| 等待 | waiting | 等待 | 用户长时间未操作 |

### 3.2 映射规则

1. **一个后端动作对应一个精灵图动画**：通过 `animation_name` 字段配置
2. **多个后端动作可复用同一动画**：如 feed 和 clean 都用 `waving`
3. **新增后端动作只需选动画**：在 `pet_action` 表插入记录，`animation_name` 填已有的 sprite key
4. **sprite 动画不够用时**：同一个动画可以映射多个动作，不影响功能

### 3.3 扩展示例

假设要新增"跳舞"动作：

```sql
INSERT INTO pet_action (action_code, action_name, animation_name, duration, 
  happiness_change, energy_change, experience_reward, bubble_text)
VALUES ('dance', '跳舞', 'jumping', 2000, 20, -10, 12, '看我跳舞~');
```

只需要指定 `animation_name = 'jumping'`，复用已有的跳跃动画，不需要改精灵图。

## 4. 前端渲染流程

```
用户点击"喂食"
    ↓
DesktopPet.runAction('feed')
    ↓
POST /pet/interact/feed → 后端处理属性变化
    ↓
返回响应（包含 animation_name = 'waving'）
    ↓
DesktopPet 设置 currentAction = 'waving'
    ↓
PetSprite 接收 action='waving'
    ↓
在 sprite sheet 中找到 row=3, frames=4
    ↓
播放 waving 动画（4帧，不循环）
    ↓
动画结束后回到 idle
```

### 4.1 PetSprite 组件

- 接收 `kind`（宠物类型）、`action`（当前动作）、`spriteConfigProp`（sprite 配置）
- 从配置中查找 `action` 对应的行号和帧数
- 用 requestAnimationFrame 驱动帧切换
- 支持循环播放（idle/running）和单次播放（waving/jumping）

### 4.2 配置加载流程

```
DesktopPet 挂载
    ↓
loadPets() → 获取宠物列表
    ↓
loadSpriteConfig() → GET /pet/sprite/config/{petTypeId}
    ↓
返回 {cellW, cellH, fps, spriteImage, actions}
    ↓
传给 PetSprite 的 spriteConfigProp
    ↓
PetSprite 用配置渲染动画
```

## 5. hatch-pet 接入流程

### 5.1 生成精灵图

使用 hatch-pet 插件生成宠物精灵图，输出目录结构：

```
hatch-pet-{name}/
├── decoded/          ← 各状态单独图片
├── frames/           ← 各状态各帧单独图片
├── final/
│   ├── spritesheet.png   ← 合并后的精灵图
│   ├── spritesheet.webp  ← webp 格式
│   └── validation.json   ← 布局配置
```

### 5.2 导入步骤

1. **复制精灵图文件**
   ```
   源：hatch-pet-{name}/final/spritesheet.webp
   目标：ruoyi-ui/public/img/pet/{name}.webp
   ```

2. **打开管理页面**：后台 → 宠物管理 → 精灵图配置

3. **点击"导入hatch-pet"**

4. **填写表单**：
   - 宠物类型：选择对应类型（如猫咪）
   - 精灵图路径：`/img/pet/{name}.webp`
   - validation.json：粘贴 `final/validation.json` 的内容

5. **点击导入**：系统自动解析并创建配置

### 5.3 validation.json 格式

系统支持两种格式：

**格式一：cells（扁平数组）**
```json
{
  "width": 1536,
  "height": 1872,
  "cells": [
    {"state": "idle", "row": 0, "column": 0, "used": true},
    {"state": "idle", "row": 0, "column": 1, "used": true},
    ...
  ]
}
```

**格式二：rows（嵌套数组）**
```json
{
  "width": 1536,
  "height": 1872,
  "rows": [
    {"state": "idle", "frames": ["00.png", "01.png", ...]},
    ...
  ]
}
```

## 6. 多宠物类型支持

每种宠物类型可以有独立的精灵图配置：

| 宠物类型 | 精灵图 | 格子尺寸 | 动作数 |
|----------|--------|----------|--------|
| 猫咪 | nailong.webp | 192×208 | 9 |
| 狗狗 | （待接入） | - | - |
| 兔子 | （待接入） | - | - |

- 不同宠物可以有不同的 sprite sheet
- 格子尺寸、帧率、动作列表都可以不同
- 后端动作是共享的，但每种宠物播放自己的 sprite 动画

## 7. 配置管理

### 7.1 管理页面功能

- **列表查看**：显示所有精灵图配置，包括宠物类型、预览图、格子尺寸、动作数
- **新增配置**：手动填写表单，配置精灵图路径和动作列表
- **编辑配置**：修改已有配置，支持动态增删动作行
- **导入配置**：上传 hatch-pet validation.json 自动解析
- **删除配置**：删除不需要的配置

### 7.2 动作编辑器

在新增/编辑表单中，可以动态管理动作列表：

| 动作Key | 行号 | 帧数 | 循环 |
|---------|------|------|------|
| idle | 0 | 6 | ✅ |
| running | 7 | 6 | ✅ |
| waving | 3 | 4 | ❌ |
| jumping | 4 | 5 | ❌ |

- **添加动作**：点击"添加动作"按钮
- **删除动作**：点击行尾删除按钮
- **修改属性**：直接编辑表格中的值

### 7.3 回退机制

如果数据库中没有精灵图配置，系统自动回退到 `spriteConfig.js` 默认配置（64×64 占位图）。

## 8. 技术细节

### 8.1 API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /pet/sprite/list | GET | 查询配置列表 |
| /pet/sprite/{id} | GET | 查询单个配置 |
| /pet/sprite/config/{petTypeId} | GET | 按宠物类型获取配置 |
| /pet/sprite | POST | 新增配置 |
| /pet/sprite | PUT | 修改配置 |
| /pet/sprite/{ids} | DELETE | 删除配置 |
| /pet/sprite/import | POST | 导入 hatch-pet 配置 |

### 8.2 前端组件

| 组件 | 文件 | 职责 |
|------|------|------|
| PetSprite | DesktopPet/PetSprite.vue | 精灵图帧动画渲染 |
| DesktopPet | DesktopPet/index.vue | 桌面宠物主组件，加载配置 |
| spriteConfig | DesktopPet/spriteConfig.js | 默认 fallback 配置 |

### 8.3 安全配置

Spring Security 已放行图片文件访问：

```java
.antMatchers(HttpMethod.GET, "/**/*.png", "/**/*.jpg", "/**/*.jpeg", 
             "/**/*.gif", "/**/*.webp", "/**/*.svg").permitAll()
```

## 9. 注意事项

1. **sprite 图片必须放在 `public/img/pet/` 目录**：浏览器通过 HTTP 访问，不能用本地文件路径
2. **后端重启后需要重新编译**：修改 Java 代码后需要重启 Spring Boot
3. **前端通过 Vue dev server 访问**：不要直接访问 Spring Boot 的 8080 端口
4. **hatch-pet 的 webp 格式兼容性好**：推荐使用 webp 而非 png，文件更小
5. **动画结束后自动回到 idle**：非循环动画播放完毕后回到待机状态
