# 宠物小游戏重构规范

## Why

现有三个游戏（猜数字、石头剪刀布、掷骰子）玩法过于简单，缺乏策略性和视觉趣味性，且都是一次性体验，没有关卡递进和成长感。需要重构为带关卡系统、道具系统的多层游戏体验，提升可玩性和留存感。

## What Changes

* **删除**现有的猜数字、石头剪刀布、掷骰子三个游戏

* **重构**游戏菜单为数据驱动模式（gameList 配置数组）

* **新增**记忆翻牌游戏（memory）- 带3个关卡，难度递增

* **新增**打地鼠游戏（whack）- 带3个关卡，难度递增

* **新增**游戏道具系统 - 6种可在游戏中使用的道具

* **新增**关卡选择界面 - 每个游戏显示关卡进度和星级

* **优化**游戏菜单、结果展示、动画效果

## Impact

* Affected code: `src/views/components/PetGame.vue`

* 组件接口不变（props, events 保持兼容）

* 道具数据存储在 localStorage（前端独立，不依赖后端接口）

***

## ADDED Requirements

### Requirement: 关卡系统

每个游戏 SHALL 包含 3 个递进关卡，难度逐级提升。

#### 记忆翻牌关卡设计

| 关卡  | 卡牌数量     | 网格  | 图案种类 | 时间限制 | 3星条件 | 2星条件 | 1星条件 |
| --- | -------- | --- | ---- | ---- | ---- | ---- | ---- |
| 第1关 | 12张(6对)  | 3×4 | 6种   | 60秒  | ≤14步 | ≤20步 | 通关   |
| 第2关 | 16张(8对)  | 4×4 | 8种   | 90秒  | ≤18步 | ≤26步 | 通关   |
| 第3关 | 20张(10对) | 4×5 | 10种  | 120秒 | ≤24步 | ≤32步 | 通关   |

* 每关星级记录保存到 localStorage

* 需要前一关至少1星才能解锁下一关

* 通关后显示星级评价和用时

#### 打地鼠关卡设计

| 关卡  | 时长  | 地鼠停留   | 同时出现 | 目标分数 | 3星    | 2星    | 1星    |
| --- | --- | ------ | ---- | ---- | ----- | ----- | ----- |
| 第1关 | 30秒 | 1500ms | 1只   | 50分  | ≥80分  | ≥60分  | ≥40分  |
| 第2关 | 30秒 | 1200ms | 1-2只 | 80分  | ≥120分 | ≥100分 | ≥80分  |
| 第3关 | 25秒 | 900ms  | 1-3只 | 100分 | ≥160分 | ≥130分 | ≥100分 |

* 第2关起会出现"炸弹地鼠"（💣），点击扣15分

* 第3关会出现"金色地鼠"（⭐），点击额外+15分

* 需要前一关至少1星才能解锁下一关

#### Scenario: 关卡解锁

* **WHEN** 玩家在某关卡获得至少1星评价

* **THEN** 下一关卡解锁，可进入挑战

#### Scenario: 关卡选择

* **WHEN** 玩家选择一个游戏

* **THEN** 显示关卡选择界面，包含每关的星级状态和锁定状态

***

### Requirement: 游戏道具系统

系统 SHALL 提供 6 种游戏内道具，玩家可在游戏过程中主动使用以获得优势。

#### 道具列表

| 道具ID   | 名称   | 图标  | 适用游戏 | 效果描述               | 获取方式            |
| ------ | ---- | --- | ---- | ------------------ | --------------- |
| hint   | 提示之眼 | 👁️ | 记忆翻牌 | 自动翻开一对匹配的卡牌（3秒后翻回） | 通关奖励/每5分钟免费获得1个 |
| freeze | 时间冻结 | ⏸️  | 打地鼠  | 暂停倒计时5秒            | 通关奖励/每5分钟免费获得1个 |
| shield | 护盾   | 🛡️ | 打地鼠  | 下一次点击炸弹不扣分         | 通关奖励/每5分钟免费获得1个 |
| peek   | 透视镜  | 🔍  | 记忆翻牌 | 所有卡牌短暂展示正面1.5秒     | 2星通关奖励          |
| slow   | 减速器  | 🐌  | 打地鼠  | 地鼠停留时间延长50%，持续10秒  | 2星通关奖励          |
| double | 双倍卡  | ✨   | 通用   | 本局获得的经验翻倍          | 3星通关奖励          |

#### 道具获取规则

* 每通关1关获得：随机1个基础道具（hint 或 freeze）

* 2星通关额外获得：peek 或 slow（随机）

* 3星通关额外获得：1个 double

* 每5分钟自动恢复：1个 hint + 1个 freeze（上限各5个）

* 道具数量存储在 localStorage

#### 道具使用规则

* 每局游戏最多使用 2 个道具

* 游戏界面右侧显示道具栏，点击道具图标使用

* 使用后道具数量-1，显示使用动画

* 已使用的道具在当局变灰不可再用

#### Scenario: 使用提示之眼

* **WHEN** 玩家在记忆翻牌游戏中点击 hint 道具

* **THEN** 随机翻开一对未匹配的相同卡牌，展示3秒后自动翻回

#### Scenario: 使用时间冻结

* **WHEN** 玩家在打地鼠游戏中点击 freeze 道具

* **THEN** 倒计时暂停5秒，地鼠继续出现和消失

#### Scenario: 使用护盾

* **WHEN** 玩家在打地鼠游戏中点击 shield 道具

* **THEN** 激活护盾状态，下一次点击炸弹地鼠时不扣分，护盾消失

#### Scenario: 使用透视镜

* **WHEN** 玩家在记忆翻牌游戏中点击 peek 道具

* **THEN** 所有卡牌短暂展示正面1.5秒后翻回，玩家需记住位置

#### Scenario: 使用减速器

* **WHEN** 玩家在打地鼠游戏中点击 slow 道具

* **THEN** 接下来10秒内地鼠停留时间延长50%

#### Scenario: 使用双倍卡

* **WHEN** 玩家在任意游戏中点击 double 道具

* **THEN** 本局获得的经验翻倍

### Requirement: 游戏菜单优化

系统 SHALL 提供优化后的游戏选择菜单：

* 卡片悬停时有更丰富的动画效果

* 显示游戏难度标签

* 显示每个游戏的关卡进度（如 ⭐⭐☆ / ⭐⭐⭐）

* 显示最佳成绩

* 新游戏标记

### Requirement: 关卡选择界面

选择游戏后，进入关卡选择界面：

* 显示3个关卡卡片，排列方式为水平或网格

* 已通关关卡显示星级（0-3星）

* 未解锁关卡显示锁定图标

* 点击已解锁关卡进入游戏

* 显示当前持有的道具数量摘要

### Requirement: 游戏内界面

游戏进行中，界面包含：

* 左侧/主区域：游戏画面

* 右侧/底部：道具栏（最多显示6个道具，灰色表示不可用/已用完）

* 顶部：关卡信息（第X关）、分数/步数、计时器

* 使用道具的反馈动画

### Requirement: 游戏结果界面

游戏结束后，结果界面包含：

* 星级评价（1-3星，用大星星动画展示）

* 本次得分/用时

* 获得的经验值

* 获得的道具奖励（图标+数量）

* 宠物表情反应（根据星级：1星😐 2星😊 3星🎉）

* "再玩一次"按钮

* "下一关"按钮（仅当解锁了下一关时显示）

* "返回菜单"按钮

## MODIFIED Requirements

### Requirement: 组件接口

组件保持现有接口不变：

* props: visible, petName, petId

* events: update:visible, finish

* finish 事件参数: { score: Number, message: String }

### Requirement: 奖励机制

每个关卡的基础经验奖励：

* 记忆翻牌：1星+10，2星+25，3星+50

* 打地鼠：1星+10，2星+25，3星+50

* 使用 double 道具后经验翻倍

## REMOVED Requirements

### Requirement: 猜数字游戏

**Reason**: 玩法过于简单，缺乏策略性和视觉趣味
**Migration**: 替换为记忆翻牌游戏

### Requirement: 石头剪刀布游戏

**Reason**: 纯运气成分，交互单一
**Migration**: 替换为打地鼠游戏

### Requirement: 掷骰子游戏

**Reason**: 纯运气成分，与石头剪刀布重复性高
**Migration**: 替换为打地鼠游戏

## 可扩展架构设计

### 游戏配置驱动模式

游戏菜单由 `gameList` 数组驱动渲染：

```js
gameList: [
  {
    key: 'memory',
    name: '记忆翻牌',
    icon: '🃏',
    desc: '翻牌配对消除',
    levels: [
      { id: 1, name: '新手入门', star: 0, unlocked: true },
      { id: 2, name: '进阶挑战', star: 0, unlocked: false },
      { id: 3, name: '大师之路', star: 0, unlocked: false },
    ],
    difficulty: 'medium',
    isNew: true,
  },
  // ...后续添加新游戏只需在此数组追加一项
]
```

### 后续添加新游戏的标准步骤

1. 在 `gameList` 数组中添加游戏配置对象
2. 在 `data()` 中添加 `{key}Game` 状态对象
3. 在 `resetAllGames()` 中添加重置逻辑
4. 在 template 中添加 `v-if="currentGame === '{key}'"` 的游戏 UI 区域
5. 在 methods 中实现游戏核心逻辑
6. 在 `finishGame()` 中添加奖励计算分支
7. 在道具配置中标明适用游戏范围

## ADDED Requirements (交互优化)

### Requirement: 游戏菜单交互优化
- 卡片添加入场动画（stagger 动画，依次淡入上移）
- 悬停时卡片放大并显示阴影
- 显示关卡进度条（如 2/3 关已完成）
- 显示道具库存小图标

### Requirement: 关卡选择交互优化
- 关卡卡片从左到右排列，带连线
- 已通关关卡有发光边框
- 未解锁关卡有锁链动画
- 悬停显示关卡详情（目标分数、3星条件）

### Requirement: 游戏过程交互优化
- 道具使用时全屏短暂闪烁效果
- 连击/连错时显示飘字动画（+10、MISS）
- 关卡切换时有过渡动画
- 记忆翻牌匹配成功时卡牌发出光晕
- 打地鼠击中时地鼠出现爆炸粒子效果

### Requirement: 结果页面交互优化
- 星级逐个弹出动画（间隔300ms）
- 道具奖励有掉落动画
- 宠物表情有弹跳动画
- 经验值数字滚动效果

## ADDED Requirements (性能优化)

### Requirement: 性能优化
系统 SHALL 保证游戏运行流畅，不出现卡顿、加载缓慢等问题。

#### 渲染性能优化
- 使用 `v-show` 替代 `v-if` 切换频繁显示的元素（如卡牌、地鼠格子）
- 卡牌翻转使用 CSS transform + will-change 优化，避免重排
- 地鼠动画使用 CSS animation 而非 JavaScript 定时器驱动
- 道具栏使用 `v-memo` 或 `key` 优化列表渲染

#### 动画性能优化
- 所有动画使用 transform 和 opacity 属性（GPU 加速）
- 避免在动画中触发 layout/paint（不修改 width/height/top/left）
- 使用 requestAnimationFrame 替代 setInterval 进行倒计时
- 粒子效果使用 CSS 实现，不引入额外库

#### 内存管理优化
- 游戏结束时清除所有定时器（setInterval/setTimeout）
- 组件销毁时（dialog 关闭）清除所有定时器和事件监听
- 避免在 data 中存储大量不必要的响应式数据
- 卡牌图案使用 emoji 而非图片，减少资源加载

#### localStorage 优化
- 读写 localStorage 时使用 try-catch 防止异常
- 道具定时恢复使用时间戳比较，而非持续运行定时器
- 减少 localStorage 读写频率，批量操作

#### 定时器管理
- 所有定时器 ID 存储在 data 中，便于清除
- 使用 `this.$once('hook:beforeDestroy', ...)` 确保定时器清理
- 游戏暂停/退出时立即清除所有活跃定时器
